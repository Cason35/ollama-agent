# 最新技术路线（Ollama 版）

## 当前主栈
- 前端：Next.js + React + TypeScript
- 模型服务：Ollama
- 主模型：`qwen2.5:14b`
- 后续可选模型：`qwen2.5-coder:14b`
- 本地接口：
  - Ollama 原生：`http://localhost:11434/api/chat`
  - OpenAI 兼容：`http://localhost:11434/v1`

## 为什么先用 Ollama
因为：
1. 本地可跑，不需要先充值 API
2. 足够学习 Agent 基础
3. 可以练前端产品能力和工程能力
4. 后面如果切换到 OpenAI API，很多代码和思路可以复用

## 要注意的限制
Ollama 的 OpenAI 兼容 `responses` 接口当前是**非状态化**的，因此：
- 不支持 `previous_response_id`
- 聊天历史需要自己维护
- 一些云端平台内置工具和 trace 能力暂时没有

这不影响前 1-2 个月的学习。

# 第 3 天要做的事情（下一步）

## 第 3 天主题
把“页面上显示聊天记录”升级成“模型真正理解上下文的多轮对话”。

## 第 3 天目标
实现真正的上下文聊天，而不只是前端显示历史消息。

## 为什么这是下一步
因为当前版本虽然像聊天产品，但如果每次后端只把当前这句用户输入发给模型，那么模型实际上并不知道前面说过什么。

所以第 3 天要解决的是：

**让完整历史消息真正参与模型推理。**

---

## 第 3 天核心任务

### 任务 1：前端改造
前端发送请求时，不再只传：

```json
{ "message": "你好" }
```

而是传完整历史：

```json
{
  "messages": [
    { "role": "user", "content": "你好" },
    { "role": "assistant", "content": "你好，有什么可以帮你？" },
    { "role": "user", "content": "请继续解释 Agent" }
  ]
}
```

### 任务 2：后端改造
后端接收完整 `messages` 后，直接转发给 Ollama：

- `model: "qwen2.5:14b"`
- `messages`
- `stream: false`

### 任务 3：验证真正上下文
测试多轮对话是否成立：

第一句：
> 我叫 Cason，你先记住

第二句：
> 我刚刚叫什么名字？

如果模型能回答出 `Cason`，说明上下文已接通。

---

## 第 3 天验收标准
满足以下条件即为通过：

1. 前端发送的是完整 `messages`
2. 后端接收并转发完整 `messages`
3. 模型能根据上一轮对话回答当前问题
4. 页面仍能正常显示聊天记录
5. 请求链路不报错

---

## 第 3 天建议顺手优化
可一并补这两项：

1. 发送后清空输入框
2. loading 状态下禁止重复点击发送

# 当前关键认知总结

到目前为止，最重要的一条认知是：

## UI 连续 ≠ 上下文连续

- 页面保存聊天记录，只是“显示上的连续”
- 只有把完整历史消息传给模型，才是“真正的上下文连续”

这一点是后续做：
- 多轮对话
- Tool calling
