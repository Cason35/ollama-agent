## 🧠 问题分析（关键收获）

### 1️⃣ Streaming 本质
- 后端：返回流（response.body）
- 前端：逐块读取（reader.read）
- UI：不断更新内容

👉 本质是“边生成边渲染”

---

### 2️⃣ 多轮对话卡顿原因（重要）

可能原因：

- messages 越来越大（上下文无限增长）
- 每次请求带全部历史
- 前端频繁 setState

👉 这是典型问题：**上下文膨胀（Context Explosion）**

# 🚀 第5天学习计划

## 🎯 主题：JSON结构化输出（进入Agent核心）

---

## 🧠 为什么要学

现在你的模型：

👉 只能“聊天”

目标：

👉 让模型“按规则输出” → 可被程序解析

---

## 🧩 第5天核心任务

### 任务1：让模型输出 JSON

示例：

```json
{
  "action": "search",
  "keyword": "北京天气"
}
```

---

### 任务2：后端解析 JSON

```ts
JSON.parse(modelOutput)
```

---

### 任务3：前端根据 JSON 执行逻辑

例如：

- action=search → 调用搜索
- action=chat → 正常聊天

---

## 🧪 验收标准

- [ ] 模型能稳定输出 JSON
- [ ] 后端成功解析 JSON
- [ ] 前端根据 JSON 执行动作
- [ ] 无解析错误

---

## ⚡ 可选优化

- JSON 校验（try/catch）
- fallback（解析失败兜底）
- prompt 优化（提高稳定性）
