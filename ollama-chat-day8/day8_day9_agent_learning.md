### 🧠 核心收获

#### 1️⃣ 上下文不是越多越好

-   现在：只传"必要信息" ✅

👉 本质： LLM 是有限上下文窗口系统，必须人为管理

------------------------------------------------------------------------

#### 2️⃣ Context Explosion（上下文膨胀）

你已经亲自踩过这个坑： - 消息越来越多 - 请求越来越慢 - UI 越来越卡

👉 解法： - 限制长度 - 裁剪历史 - 控制 token

------------------------------------------------------------------------

#### 3️⃣ "记忆"是可以设计的

你现在已经实现： 👉 保留"姓名"

说明： Agent 的记忆 ≠ 聊天记录\
而是"被设计过的信息"

### Todo 工具

``` ts
function generateTodos(text: string) {
  return [
    "1. 分析需求",

    "3. 测试功能"
  ];
}
```

## Step 3：后端分发逻辑

``` ts
switch (parsed.action) {
  case "weather":
    return realWeather(parsed.keyword);

  case "summary":
    return summarize(parsed.content);

  case "todo":
    return generateTodos(parsed.content);

  default:
    return chat;
}
```
