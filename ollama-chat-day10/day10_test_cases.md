# Day10 测试用例

## 1) 正常场景：总结能力

- 输入：`帮我总结一下刚刚内容`
- 预期：
  - 返回 `type: "summary"`
  - 文本包含 3-5 条要点
  - 与最近对话相关
- 验证点：
  - 前端总结卡片正常渲染
  - 控制台可见 `[Agent] route` 与 `[Agent] result`

## 2) 正常场景：待办能力

- 输入：`帮我把今天的学习拆成待办`
- 预期：
  - 返回 `type: "todo"`
  - `items` 是数组，至少 3 项
  - 每项包含 `task` 和 `done`
- 验证点：
  - 前端展示复选框列表
  - `done` 默认 `false`

## 3) 边界场景：空输入 / 很短输入

- 输入：空字符串（前端不会提交）
- 预期：不发请求、不崩溃
- 输入：`总结`
- 预期：仍可返回 summary，接口稳定

## 4) 异常场景：todo JSON 解析失败

- 场景：模型返回非 JSON 或混杂文本
- 预期：
  - 后端会尝试直接解析 + 提取 `[]` 再解析
  - 失败时触发 fallback 待办列表
  - 接口不崩溃

## 5) 可观测性检查

- 每次调用确认日志字段齐全：
  - action
  - input
  - durationMs
  - success
